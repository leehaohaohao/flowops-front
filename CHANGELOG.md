# Changelog

## 2.4.0 (2026-06-04)

### 优化

- 项目列表从表格改为卡片网格布局，支持响应式三栏/两栏/单栏排列
- 项目卡片展示服务数、运行中服务数、成员数统计
- 新增"上次访问"快捷入口，点击项目卡片时自动记录

---

## 2.3.0 (2026-05-31)

### 新功能

- 服务编辑表单支持运行时语言选择（Java / Go），根据选择动态切换表单字段
- 上传组件支持拖拽上传文件
- Go 运行时表单自动适配：隐藏 JAR 相关字段，显示二进制上传入口

### 优化

- ServiceEdit 表单布局优化为双栏结构，提升表单填写效率
- 编辑模式下正确回显运行时语言，不再丢失已选值

### 修复

- 修复 nginxListenPort JSX 中多余的 `>` 符号
- 修复编辑模式下运行时选择丢失，表单回显为 Java 默认值

---

## 2.2.0 (2026-05-28)

### 新功能

- 端口映射组件重构，支持 host / expose / multiple 三种端口映射类型
- 服务表单新增 `deployName` 字段（Docker Compose 项目命名）
- 服务表单新增 `remark` 备注字段

### 优化

- `port_mappings` JSON 字段替代原 `port` / `extraPorts`，统一端口配置模型
- fullstack / frontend 模式支持自定义 nginx 端口，不再硬编码 listen 80

### 移除

- 移除旧 `port` / `extraPorts` 字段及所有相关表单引用

---

## 2.1.0 (2026-05-27)

### 新功能

- 服务编辑表单新增 `extraPorts` 额外端口映射配置，支持多端口映射
- Docker Compose 生成时自动追加额外端口映射配置

---

## 2.0.0 (2026-05-24)

### 新功能

- 前端从 Thymeleaf 模板迁移为 React SPA（Ant Design + React Router）
- 实现项目级权限隔离，每个项目独立管理成员和角色
- 新增用户管理页面，支持创建/编辑用户及多项目角色分配
- 新增成员管理页面，对接 `PUT /api/users/{id}` 统一更新接口和 `GET /api/users/assignable` 接口
- 新增跨项目授权管理（ProjectAccess）
- 新增容器运行日志查看页面，支持实时日志跟踪
- 新增部署日志页面
- 项目列表新增服务数统计列
- 适配后端项目组合并变更，移除 groups 概念

### 优化

- 权限校验改为基于权限码（permission code）判断，不再依赖角色名硬编码
- 主管自动拥有管辖项目全部权限，操作按钮正确显示
- 成员管理中禁止对自己修改角色或移除
- 所有接口对接类型化 DTO/VO 返回值
- 环境变量统一从 `.env` 文件读取，支持多环境切换
- 路由切换为 HashRouter

### 修复

- UserContext 空值守卫，防止 userInfo 为 null 时页面崩溃
- UserInfo 字段 `isSuperAdmin` → `superAdmin`，适配后端序列化格式
- 用户列表角色列根据 isSuperAdmin 显示超级管理员标签
- 权限复选框使用 assignable 接口返回的可分配权限渲染
- 编辑用户时正确预填充角色和权限复选框
- 容器日志路由修正为 `projects/:projectId/services/:id/logs`

---

## 1.1.0 (2026-05-22)

### 新功能

- 搭建前端项目脚手架（React + Ant Design + React Router）
- 用户登录认证页面
- Dashboard 统计面板
- 服务管理页面：对接后端服务列表接口，支持创建、编辑、删除部署服务
- 服务编辑页面：对接后端创建/更新/上传接口
- 容器状态查看和日志查看
- Axios 请求封装，统一错误处理

### 优化

- 简化页面设计，去除渐变和装饰元素，回归功能性 UI
- 后端地址统一从环境变量读取
