# ServiceEdit 表单布局优化

## Context
ServiceEdit.tsx 当前是纯纵向堆叠布局，所有字段从上往下排列，导致页面很长。上传产物区在最底部，需要滚动很久才能看到。需要优化布局，让页面更紧凑、信息密度更高。

## 方案：双栏布局 + 内部网格化

### 整体结构：左右双栏

```
┌─ 标题 + 操作按钮（保存/预览/取消）─────────────────────┐
├──────────────────────────┬────────────────────────────┤
│                          │                            │
│   左栏（表单，~60%）      │   右栏（侧边栏，~40%）      │
│                          │                            │
│   基本信息（3列网格）      │   ┌─ 上传产物（编辑模式）──┐ │
│   端口映射（紧凑表格）     │   │  后端产物拖拽上传      │ │
│   后端配置 Card           │   │  前端产物拖拽上传      │ │
│   前端配置 Card           │   └──────────────────────┘ │
│                          │                            │
│                          │   ┌─ 配置预览 ─────────────┐ │
│                          │   │  Dockerfile            │ │
│                          │   │  docker-compose.yml    │ │
│                          │   └──────────────────────┘ │
│                          │                            │
└──────────────────────────┴────────────────────────────┘
```

### 关键改动

1. **标题栏**：标题左侧，操作按钮（保存/预览/取消）右侧同行
2. **左栏 - 基本信息**：服务类型 + 服务名称 + 部署名称 + 备注，用 Row/Col 网格排列（3列一行），不单独占行
3. **左栏 - 端口映射**：保持水平列表，但去掉外层 Form.Item 的 label 冗余，更紧凑
4. **左栏 - 后端配置**：
   - Card 内第一行：运行时（Select）独占一行或与数据持久化并排
   - **高级选项折叠区（Collapse）**：基础镜像、启动命令（选择运行时后自动填充，大多数情况不需要手动改）
   - 环境变量：2列网格
   - 数据持久化：2列
5. **左栏 - 前端配置**：
   - Card 内第一行：运行时（Select）+ 后端地址（纯前端模式）
   - **高级选项折叠区（Collapse）**：基础镜像、Nginx 监听端口、自定义 Nginx 配置（合并到同一个折叠区）
   - 代理规则：保持当前结构
6. **右栏 - 上传产物**：从底部移到右侧 sticky 侧边栏，编辑模式下始终可见
7. **右栏 - 预览区**：预览结果直接显示在右侧侧边栏（替代 Modal 弹窗），实时更新
8. **右侧栏 sticky**：`position: sticky; top: 16px`，滚动时始终可见

### 修改文件
- `src/pages/ServiceEdit.tsx` — 唯一需要修改的文件

### 技术细节
- 使用 Ant Design 的 `Row` / `Col` 组件做网格布局
- 右侧栏用 CSS `position: sticky` 实现固定
- 预览从 Modal 改为右侧栏内直接渲染，通过 `generatePreview` 实时计算
- 所有字段保留，不删除任何功能
- 响应式：左栏 `span={14}`，右栏 `span={10}`

### 验证
- 创建服务页面：检查布局是否为双栏，基本信息是否紧凑
- 编辑服务页面：检查上传产物是否在右侧可见，无需滚动
- 纯后端/纯前端/前后端一体三种类型切换是否正常
- 预览配置是否在右侧栏正确显示
- 表单验证和保存功能不受影响
